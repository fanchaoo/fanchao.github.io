Spring-IoC

发表于 2019-01-22 | 分类于 Spring |

整个IoC有两个重要的部分：1. 让容器创建Bean；2. 让容器为Bean设置属性。

1. 概述

1.1. IoC

IoC的全称是Inverse of Control，就是“控制反转”的意思。控制反转这个概念其实包含两个层面的意思：“控制”是接口实现类的选择控制权；而“反转”是指这种控制权从调用类转移到了第三方容器的手中。

IOC的思想最核心的地方在于，资源交由第三方去集中管理，使得资源可配置，易管理。

在Spring中，可以“使用配置信息（XML配置文件，注解）来描述需要实例化的类的信息，以及对象与对象之间的依赖关系”。而Spring容器具体做的事情，其实就是读取我们设定好的“配置信息”，然后通过“Java语言的反射机制”，帮我们去统一地“创建Bean，并建立Bean之间的依赖关系”。

1.2. 反射

通过Java的反射机制，我们可以通过程序化的方式，访问目标类中的所有元素，比如构造器，字段，方法等等。

关于“反射”，有几个很重要的类：ClassLoader，Class，Constructor，Method，Field。

在Java中，可以通过一个“全限定类名的字符串”，来获取（其实就是通过类加载器来加载这个类）这个类的Class对象。有了这个对象，就可以进行各种反射操作了。

2. 使用方式

让容器创建Bean

我们需要提供一些配置信息给Spring。Spring在容器创建的时候，会读取这些配置信息，并解析成相应的BeanDefinition，最终根据这些BeanDefinition创建出Bean的实例。

  • @Bean注解
  • @ComponentScan注解 + @Component（@Controller，@Service，@Repository）注解
  • @Import注解 + ImportSelector接口 + ImportBeanDefinitionRegistrar接口
  • FactoryBean接口

带条件的创建

  • @Conditional注解：通过创建Condition接口的实现类，覆盖match方法，来自定义如何加载该bean
  • @Profile注解：通过虚拟机参数或环境变量中spring.profiles.active的值，来确定是否创建该bean

让容器为Bean设置属性

除了可以通过在构造器中设置属性值，或者在工厂方法里设置属性值，还可以通过下面方式为Bean注入属性值：

@Value

@Value注解可以为对象注入“基本类型（及其包装类型）的值”和“String类型的值”。

用法：

  • 字面量，比如@Value("tom")
  • SpringEL表达式，比如@Value("#{1 + 1}")
  • 读取环境变量中的值，比如@Value("${os.name}")，读取当前操作系统的名称
  • 配合@PropertySource注解，读取.properties配置文件中的值，比如@Value(${name})

@Autowired

@Autowired注解可以为一个对象的属性，注入实例（从容器中找出一个实例，然后注入进去）。

@Autowired也可以注入String对象，和基本类型的值及其包装类对象，但很少这么用。

@Autowired可以标注在“属性，构造器，方法，参数”上。
